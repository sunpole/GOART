# GOART — Рабочий день в типографии

**GOART** — русскоязычная браузерная игра-симулятор рабочего дня в типографии. Игрок перемещается по карте локаций, встречает NPC, получает задания, выполняет рабочие действия и постепенно проходит цепочку событий до финального взаимодействия с боссом.

Проект сделан как лёгкая HTML/CSS/JavaScript-игра без серверной части. Основной вариант запуска — GitHub Pages.

- Сайт: https://sunpole.github.io/GOART/
- Репозиторий: https://github.com/sunpole/GOART
- Язык проекта: русский
- Основная технология: HTML, CSS, Vanilla JavaScript

## Игровая идея

Игрок оказывается в типографии во время рабочего дня. Его задача — пройти рабочий маршрут, выполнить поручения, взаимодействовать с коллегами и завершить день.

Основная петля игры:

1. Игрок начинает в офисе.
2. По карте перемещаются NPC.
3. Игрок встречает персонажей.
4. Персонажи запускают диалоги, подсказки, препятствия или квесты.
5. Игрок выполняет задания: например, цветопробу и лак.
6. После выполнения основных задач игрок может попасть к боссу.
7. Финал проходит через блиц-викторину по печатной тематике.

## Карта локаций

Карта описана массивом `ROOMS` в `main.js`.

Сейчас в игре используются локации:

- Офис
- Коридор
- Склад 1 эт
- ПР. цех дверь
- Аквариум
- Коридор 2
- Дверь на 2 эт
- Проход в произв.
- Производство
- Лак
- Паллеты
- Склад 2 эт
- Кабинет босса

Каждая локация имеет:

- `id` — уникальный идентификатор;
- `name` — название;
- `icon` — эмодзи;
- `doors` — список соседних комнат;
- `desc` — описание;
- `limit` — максимальное количество людей в комнате.

Текущие персонажи:

- Катя — клиентский менеджер, связана с квестом цветопробы;
- Светлана — персонаж-препятствие, задерживает игрока разговором;
- Владимир — нейтральный персонаж;
- Сергей Ас — допечатник, даёт тематические фразы;
- Саша Ха — позитивный персонаж, снижает стресс;
- Марина — главная по цифре, может отбирать предметы;
- Арсений — помогает игроку;
- Виктор — босс и финальный персонаж.

У NPC есть:

- имя;
- иконка;
- портрет;
- описание;
- тип;
- домашние локации;
- место появления;
- интервал движения;
- диалоги с вероятностями.

## Квесты

В проекте сейчас выделены три основные задачи:

1. **Цветопроба** — сделать цветопробу для Кати.

## Известные проблемы

На момент подготовки документации обнаружены следующие технические проблемы:

1. В `resetGame()` есть обращение к элементу `#player-name`, которого нет в текущем `index.html`. Это вызывает ошибку вида:

```text
Uncaught TypeError: Cannot set properties of null (setting 'innerText')
```

2. Функция `shuffle()` определена в `main.js` дважды. Это не всегда ломает запуск, но усложняет поддержку и может привести к путанице.

3. Функция `legendOpen()` перезаписывает содержимое `#legendCont` через `innerHTML`, из-за чего может исчезать кнопка закрытия справки.

4. В проекте используются портреты NPC из папки `img/`, но наличие всех изображений нужно отдельно проверить.

5. Вся логика пока находится в одном большом `main.js`; в будущем желательно разделить код на модули.

См. также: `docs/KNOWN_BUGS.md`.

## GOART и AWB

В рамках обсуждения проекта появилась отдельная идея **AWB — Adaptive Web Browser / Адаптивный веб браузер**.

AWB — это не сама игра, а потенциальная Android WebView-обёртка, которая сможет открывать GOART или другие веб-страницы в управляемом мобильном режиме.

Идея AWB:

- открыть GOART внутри Android-приложения;
- дать настройки URL;
- поддержать переключаемый landscape;
- добавить настройки масштаба, прокрутки и выделения текста;
- тестировать отображение на старых Android-устройствах, включая Samsung Galaxy Note 5.

Подробные заметки находятся в `PROJECT_NOTES_AWB_GOART.md`.
