# Мобильная стратегия GOART и идея AWB

Дата фиксации: 2026-06-13.

Документ описывает мобильную стратегию GOART и отдельную идею AWB — Adaptive Web Browser.

---

## 1. Почему мобильная стратегия важна

GOART размещается на GitHub Pages и должен открываться как обычный веб-проект.

При этом в обсуждении отдельно рассматривался старый телефон Samsung Galaxy Note 5. Для таких устройств важно понимать разницу между физическим разрешением экрана и CSS viewport.

Телефон может иметь физическое разрешение около 2560×1440, но браузер обычно отдаёт сайту логическую ширину примерно 360–412 CSS px. Поэтому нельзя проектировать обычный мобильный сайт так, будто ему доступна вся физическая ширина экрана.

---

## 2. Два пути развития интерфейса

### Путь 1. Обычный адаптивный веб

Это основной и самый безопасный путь.

Ориентиры:

```text
ширина: 360–412 CSS px
портретный режим
простая HTML/CSS/JS-структура
без обязательного APK
GitHub Pages как основной хостинг
```

Плюсы:

- проще тестировать;
- не нужен Android Studio;
- не нужен Play Market;
- работает на ПК и телефонах;
- проще не сломать проект.

Минусы:

- меньше места на экране;
- сложнее сделать широкую игровую карту;
- невозможно полностью контролировать браузерное поведение.

### Путь 2. APK / WebView / Landscape

Этот путь нужен, если получится собрать Android-приложение-обёртку.

Ориентиры:

```text
WebView
переключаемый landscape
возможность открыть GOART внутри приложения
управление масштабом и прокруткой
тест на Samsung Galaxy Note 5
```

Плюсы:

- больше контроля над отображением;
- можно пробовать широкую сцену;
- можно сделать приложение для запуска GOART;
- можно добавить настройки отображения.

Минусы:

- сборка APK сложнее;
- нужны Android-права, manifest, minSdk, WebView-настройки;
- можно легко заблокировать клики или прокрутку неправильными настройками;
- сложнее поддерживать.

---

## 3. Текущий вывод

Пока стабильный APK не собран, GOART нужно развивать как обычный адаптивный веб-проект.

AWB можно вести как отдельную экспериментальную ветку/проект, но не смешивать его с основной игрой.

---

## 4. Что такое AWB

AWB — Adaptive Web Browser / Адаптивный веб браузер.

Это отдельное Android WebView-приложение, которое открывает веб-страницу внутри себя.

URL по умолчанию:

```text
https://sunpole.github.io/GOART/
```

AWB должен быть не игрой, а инструментом для запуска и тестирования веб-проектов.

---

## 5. Требования к AWB

### 5.1. Основные функции

```text
1. Открывать сайт через WebView.
2. Иметь URL по умолчанию для GOART.
3. Позволять менять URL.
4. Иметь небольшое меню настроек.
5. Показывать описание приложения.
6. Показывать цели проекта.
7. Показывать версию.
8. Иметь таймер времени на странице.
9. Поддерживать светлую и тёмную тему.
10. Работать на старых Android-устройствах настолько, насколько это возможно.
```

### 5.2. Чекбоксы

Все чекбоксы выключены по умолчанию:

```text
[ ] Принудительная альбомная ориентация Landscape
[ ] Без масштаба
[ ] Без прокруток
[ ] Без выделения и копирования текстов
```

Важное требование: Landscape должен быть переключаемым, а не постоянным.

---

## 6. Предупреждения по реализации AWB

1. Нельзя блокировать все touch-события WebView без проверки: это может отключить клики по сайту.
2. Запрет прокрутки лучше делать аккуратно: через CSS/JS или отдельный режим.
3. Запрет выделения текста должен применяться после загрузки страницы.
4. Таймер нельзя обновлять в `TextView`, если этот элемент ещё не создан.
5. XML-комментарии в AndroidManifest нельзя вставлять так, чтобы ломать структуру тега.
6. Нужно проверить minSdk и targetSdk.
7. Нужно проверить поведение на Android 7 отдельно.

---

## 7. Черновое описание AWB для экрана приложения

```text
AWB (Adaptive Web Browser) — минималистичное WebView-приложение для открытия и проверки сайтов в адаптивном режиме.

Приложение создаётся для тестирования веб-интерфейсов на разных экранах, включая старые Android-устройства.

Основные возможности:
- открытие сайта по URL;
- смена URL;
- переключаемая альбомная ориентация;
- настройки масштаба, прокрутки и выделения текста;
- светлая и тёмная тема;
- таймер активности страницы.

Сайт по умолчанию: https://sunpole.github.io/GOART/
```

---

## 8. Где хранить AWB

Рекомендуется не смешивать исходники Android-приложения с текущим GOART, пока игра не стабилизирована.

Варианты:

```text
1. Отдельный репозиторий AWB.
2. Отдельная папка /awb внутри GOART, но только после стабилизации.
3. Отдельная ветка awb-experiment.
```

Лучший вариант на текущем этапе: отдельный репозиторий или отдельная ветка.
