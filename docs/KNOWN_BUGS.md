# Известные ошибки и технические долги

Документ фиксирует проблемы, которые были обнаружены при первичной проверке проекта.

## Критические ошибки

### 1. Ошибка `resetGame()` из-за отсутствующего `#player-name`

Симптом:

```text
main.js:463 Uncaught TypeError: Cannot set properties of null (setting 'innerText')
```

Причина:

`main.js` пытается выполнить:

```js
document.getElementById('player-name').innerText = n;
```

Но в текущем `index.html` нет элемента:

```html
<span id="player-name"></span>
```

Рекомендуемое исправление:

```js
function resetGame() {
  const n = prompt("Ваше имя?", "Новичок")?.trim() || "Новичок";

  const playerNameEl = document.getElementById('player-name');
  if (playerNameEl) {
    playerNameEl.innerText = n;
  }

  startGame(n);
}
```

## Средние проблемы

### 2. Дублирование функции `shuffle()`

В `main.js` функция `shuffle()` определена дважды.

Это не обязательно ломает игру, потому что второе объявление перезаписывает первое, но это плохой сигнал для поддержки кода.

Рекомендуемое исправление:

- оставить только одну функцию `shuffle()`;
- лучше оставить более документированную версию в верхней части файла;
- вторую удалить.

### 3. `legendOpen()` удаляет кнопку закрытия

Причина:

`legendOpen()` делает:

```js
document.getElementById("legendCont").innerHTML = html;
```

Это полностью заменяет содержимое контейнера справки. Если кнопка закрытия была внутри `#legendCont`, она исчезает.

Рекомендуемое исправление:

- либо добавлять кнопку закрытия в генерируемый HTML;
- либо создать отдельный контейнер для содержимого справки, например `#legendBody`, а кнопку закрытия оставить вне него.

### 4. Возможные отсутствующие изображения NPC

В NPC указаны портреты:

```text
img/katya.jpg
img/svetlana.jpg
img/vladimir.jpg
img/sergey.jpg
img/sasha.jpg
img/marina.jpg
img/arseniy.jpg
img/viktor.jpg
```

Нужно проверить, что все файлы реально существуют в репозитории и правильно названы.

Особенно важно проверить разницу:

```text
arseny.jpg
arseniy.jpg
```

В раннем обсуждении встречалось имя `arseny.jpg`, а в текущем коде указано `arseniy.jpg`.

## Архитектурные долги

### 5. Один большой `main.js`

Сейчас данные, рендер и логика находятся в одном файле.

На ранней стадии это допустимо, но для развития проекта лучше разделить код.

### 6. Нет автоматической проверки карты

Нужна функция валидации:

- все `doors` указывают на существующие комнаты;
- все `home` у NPC указывают на существующие комнаты;
- сумма вероятностей диалогов не превышает 100;
- все `spawn` и `workplace` существуют;
- все квестовые ключи есть в `gameState.player.quests`.

### 7. Нет тестового режима

Для быстрой отладки желательно добавить dev-режим:

- сразу выдать цветопробу;
- сразу выполнить лак;
- телепорт к боссу;
- показать всех NPC;
- сбросить NPC;
- отключить таймеры движения.

## Рекомендуемый порядок исправления

1. Исправить `resetGame()`.
2. Проверить запуск сайта.
3. Исправить кнопку закрытия справки.
4. Удалить дубль `shuffle()`.
5. Проверить изображения.
6. Создать стабильную точку отката.
7. После этого начинать рефакторинг.
