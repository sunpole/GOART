# Дорожная карта GOART

## Этап 0. Фиксация состояния

Статус: начато.

Задачи:

- создать резервную ветку перед документационными и техническими изменениями;
- обновить README;
- добавить папку `docs/`;
- зафиксировать известные ошибки.

## Этап 1. Стабилизация запуска

Цель: сайт должен открываться без ошибок консоли.

Задачи:

- исправить `resetGame()`;
- проверить запуск через GitHub Pages;
- проверить запуск через локальный сервер;
- убедиться, что `window.onload` корректно запускает игру;
- проверить, что все DOM-элементы существуют до обращения к ним.

## Этап 2. Проверка базового прохождения

Цель: пройти игру от старта до финала.

Задачи:

- проверить движение игрока;
- проверить лимиты комнат;
- проверить появление NPC;
- проверить движение NPC;
- проверить квест цветопробы;
- проверить квест лака;
- проверить вход к Виктору;
- проверить блиц босса;
- проверить победный финал;
- проверить стресс и инвентарь.

## Этап 3. Исправление известных проблем

Цель: убрать технические долги, которые мешают развитию.

Задачи:

- удалить дубль `shuffle()`;
- исправить `legendOpen()`;
- проверить портреты NPC;
- добавить защитные проверки DOM;
- добавить валидацию данных `ROOMS`, `NPCS_FULL`, `QUESTS`;
- добавить dev-режим для быстрой проверки.

## Этап 4. Релизная точка отката

Цель: сохранить первую стабильную версию.

Задачи:

- убедиться, что игра проходит полностью;
- создать ветку или тег стабильной версии;
- описать изменения в `CHANGELOG.md`;
- при необходимости создать GitHub Release.

## Этап 5. Улучшение интерфейса

Цель: сделать игру приятнее и удобнее.

Задачи:

- заменить `prompt()` на экран старта;
- улучшить footer;
- сделать нормальное окно справки;
- добавить адаптивные режимы для мобильного экрана;
- проверить ширины 360, 390, 412 px;
- проверить landscape;
- добавить визуальные состояния кнопок;
- улучшить модальные окна.

## Этап 6. Рефакторинг структуры

Цель: сделать код поддерживаемым.

Задачи:

- вынести CSS в `css/style.css`;
- вынести данные комнат в `js/data.rooms.js`;
- вынести данные NPC в `js/data.npcs.js`;
- вынести квесты в `js/data.quests.js`;
- вынести рендер в `js/render.js`;
- вынести логику NPC в `js/npc.logic.js`;
- вынести блиц босса в `js/boss.quiz.js`;
- оставить `js/main.js` как точку запуска.

## Этап 7. AWB / Android WebView

Цель: вернуться к идее Android-обёртки только после стабилизации web-версии.

Задачи:

- определить, нужен ли AWB внутри этого репозитория или отдельным репозиторием;
- подготовить Android Studio проект;
- настроить WebView;
- добавить URL по умолчанию `https://sunpole.github.io/GOART/`;
- добавить настройки ориентации, масштаба и прокрутки;
- протестировать на старом Android;
- проверить Samsung Galaxy Note 5.

## Главный принцип

Не переписывать всё сразу.

Правильный порядок:

```text
документация → стабилизация → точка отката → маленькие патчи → рефакторинг → AWB
```
