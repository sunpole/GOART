# GOART и AWB — собранные заметки по проекту

Дата фиксации: 2026-06-13.

Этот файл создан как единая памятка по текущему состоянию проекта GOART и по идее отдельного Android-приложения AWB, которое должно открывать GOART или любой другой сайт через WebView.

---

## 1. Что такое GOART

GOART — веб-проект / игра на русском языке с рабочим названием `Рабочий день в типографии`.

Адрес репозитория:

```text
https://github.com/sunpole/GOART
```

Адрес GitHub Pages:

```text
https://sunpole.github.io/GOART/
```

Судя по текущим файлам, GOART — это браузерная игра/интерактивная карта про рабочий день в типографии. Игрок перемещается по локациям, встречает NPC, выполняет задачи/квесты и взаимодействует с персонажами.

Файл `js/main.js` сейчас содержит только запуск:

## 3. Важные технические наблюдения по GOART

### 3.1. Сейчас проект ориентирован на мобильную адаптивность

В `index.html` используется:

```html
<meta name="viewport" content="width=device-width, initial-scale=1">
```

Также есть media queries под малые экраны:

```css
@media (max-width: 760px) { ... }
@media (max-width: 650px) { ... }
@media (max-width: 450px) { ... }
@media (max-height: 420px) { ... }
```

Это значит, что проект уже пытается подстраиваться под телефоны, но для старых устройств вроде Samsung Galaxy Note 5 надо отдельно проверять фактический CSS viewport.

### 3.2. Samsung Galaxy Note 5 и проблема масштаба

Обсуждённый телефон:

```text
Samsung Galaxy Note 5, 2015 год
Максимальное официальное обновление: Android 7
Физическое разрешение экрана: примерно 2560 × 1440
Проблема: сайты выглядят слишком крупно из-за CSS viewport, DPR и масштабирования Android/WebView.
```

Ключевая мысль: браузер обычно не отдаёт веб-странице физические 2560 пикселей ширины. Вместо этого сайт видит логическое CSS-разрешение, например около 360–412 px по ширине в портретном режиме.

## 4. Два возможных пути разработки интерфейса

В обсуждении были определены два пути.

Если получится собрать Android-приложение-обёртку, можно пробовать делать игру/интерфейс под более широкое рабочее поле:

```text

полноэкранного режима
принудительного landscape

```

### Путь 2. Обычный мобильный web

Если APK/WebView-сборка окажется сложной, тогда разумнее делать проекты под обычную мобильную ширину:

```text
около 400 px
```

Это проще, стабильнее и ближе к тому, как сайт реально открывается в мобильном браузере.

## 5. Что такое AWB

AWB — отдельная идея Android-приложения.

Название:

```text
AWB — Адаптивный веб браузер
```

Расшифровка:

```text
Adaptive Web Browser
```

Назначение:

```text
Открывать GOART или любую другую веб-страницу внутри Android-приложения через WebView.
```

Сайт по умолчанию:

```text
https://sunpole.github.io/GOART/
```

AWB не является самой игрой. Это приложение-обёртка, которое должно помочь тестировать и использовать веб-проекты на телефоне в более управляемом режиме.

Важное уточнение: принудительный Landscape должен быть не постоянным, а переключаемым.

## 9. Важные предупреждения по старому черновому Android-коду

Черновой код из обсуждения нельзя считать финальным без проверки.

Что обязательно перепроверить:

```text
1. В AndroidManifest.xml нельзя вставлять XML-комментарий прямо внутри открытого тега activity так, чтобы он ломал XML.
2. timerTextView не должен обновляться до того, как он реально найден в layout.
3. Если таймер находится только в диалоге настроек, нужно хранить время отдельно и обновлять TextView только когда диалог открыт.
4. Отключение прокрутки через setOnTouchListener может полностью заблокировать клики по сайту. Лучше делать это аккуратнее через CSS/JS или отдельный режим.
5. Запрет выделения текста через CSS-инъекцию должен выполняться после загрузки страницы.
6. WebView.evaluateJavascript доступен с Android 4.4+, значит для Android 5+ подходит.
7. Для современных Android нужно проверить поддержку mixed content, storage, file access, back navigation.
8. Для Google Play сейчас обычно нужен AAB, не только APK.
```

## 10. Что надо проверить в GOART перед упаковкой в APK

Перед упаковкой сайта в AWB/APK нужно убедиться, что сам сайт работает стабильно в обычном браузере.

Чек-лист:

```text
1. Открывается ли https://sunpole.github.io/GOART/ без ошибок.
2. Есть ли ошибки в консоли браузера.
3. Есть ли функция resetGame().
4. Нужен ли файл js/main.js, если он сейчас не подключён.
5. Все ли изображения из img/ грузятся.
6. Работают ли кнопки перемещения.
7. Работает ли справка.
8. Работают ли диалоги и eventBox.
9. Не ломается ли карта на ширине 360–412 px.
10. Не ломается ли карта в альбомном режиме на старом телефоне.
11. Нужен ли отдельный режим Full HD/landscape.
```

## 11. Возможные будущие режимы отображения

Для GOART/AWB можно предусмотреть пресеты:

```text
1. Mobile 400px — обычный мобильный режим.
2. Note 5 Portrait — старый Android, портретный режим.
3. Note 5 Landscape — старый Android, альбомный режим.
4. Full HD 1920×1080 — игровой полноэкранный режим.
5. Desktop Wide — широкий режим для ПК.
```

В веб-проекте это можно реализовать классами на body:

```css
body.view-mobile {}
body.view-note5 {}
body.view-landscape {}
body.view-fullhd {}
body.view-desktop {}
```

И переключать через JS:

```js
document.body.classList.add('view-fullhd');
```
